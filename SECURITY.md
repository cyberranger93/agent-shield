# Security Policy

## Responsible Use

agent-shield is a **defensive security tool** built for:
- Security researchers testing their own AI agents
- Engineering teams auditing their LLM pipelines
- Red teamers with explicit authorization to test target systems
- Students learning about AI security

**Do not use agent-shield to attack AI systems you do not own or have explicit written permission to test.**

## Reporting a Vulnerability

If you discover a security vulnerability in agent-shield itself, please report it responsibly:

1. **Do NOT open a public GitHub issue** for security vulnerabilities
2. Email: `yathavang1@gmail.com` with subject `[SECURITY] agent-shield vulnerability`
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

We will respond within 48 hours and provide a timeline for a fix.

## Scope

We consider the following in scope:
- Vulnerabilities in agent-shield's own code
- False negatives (known attack patterns not detected)
- Dependency vulnerabilities that affect the tool

## Out of Scope

- Rate limiting / DoS against the tool itself
- Social engineering attacks
- Issues already publicly disclosed

## Hall of Fame

We publicly acknowledge responsible disclosures in our CHANGELOG.
